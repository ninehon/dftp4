加密、安全小工具介紹

用於安全、加密的工具很多，我們選擇的原則盡量是：免費、體積小、無安全虞慮、操作不要太複雜。在便利與安全兼顧的情況下，我們在此介紹了：

  • Crypto Notepad - 文字加密軟體
  • KeePass - 密碼管理軟體
  • BlankAndSecure - 刪除隱私文件軟體
  • 隱藏檔案或資料夾 - 手動、Folder Password Lock Free、IsyLock

 

Crypto Notepad

Crypto Notepad 是一個強化加密功能的開源軟體，體積小、免安裝、介面很清爽，目前還沒有中文介面。不過基本使用很容易，英文不會造成太大的困擾。

常常看見有人把各種帳號密碼記在文字或 Word 文件，就存放在桌面上，檔案名稱還直接就是 "○○密碼.txt"。這樣的做法其實不太妥當，至少要放在比較不明顯的位置，檔名也別太直接，可以的話用密碼保護一下。

需要帳密登入的地方很多，記憶密碼已經變成重要卻又痛苦的事。那麼多密碼不太可能都靠記憶，有的密碼還會要求定時變更，所以才出現把密碼存到桌面，或一組密碼走天下的情事。是的，這樣是便利，但是不建議這樣做。比較安全的做法是存到有保護的文件。上次提到的 BitLocker 是把資料鎖起來，BitLocker 的密碼也靠記憶嗎？還是存起來？修復金鑰總不能也存到 BitLocker 保護的磁碟中吧！(保管箱備用鑰匙鎖在保管箱內？)

協助記憶密碼的工具很多，Crypto Notepad 就是簡單又好用的一個...

設置的最後一大項「Encryption」是設置加密方式，Crypto Notepad 採用的是 AES 演算法，可以設置演算法、使用金鑰大小、運算次數等，官方建議兼顧安全與效能的設置是 「SHA512 + 256 + 兩百萬次以上」，實際上預設值是「SHA1 + 256 + 1000」。

其實用預設值也可以了，要注意的是一但變更這裡的設置，加密後的檔案下次開啟時也要用這樣的設定才能開啟。因此想要利用此種方式達到混淆別人的話，千萬要記住自己的設定，否則可能會整到自己喔... 如果點擊程式提示的說明(如下圖黃色標示部分)，此說明文字就會消失，這應該是刻意隱藏：

Crypto Notepad 內建的密碼產生器

一般我們都建議使用強式密碼

強式密碼

  • 難以猜測或不含常見的完整字詞
  • 長度至少為八個字元
  • 包含大小寫字母、數字和符號
  • 不包含使用者名稱、姓名或公司名稱
  • 與先前的密碼差異很大
，至少不要使用過於簡單的密碼，或是過於常見的密碼。有時候為了想一個密碼傷透腦筋，這時候可以利用 Crypto Notepad 內建的密碼產生器來協助。

從「Tools」→「Password Generator」打開密碼產生器，如下圖：①是密碼長度，一般建議至少 8 個字元，②是一次產生幾組密碼；中間常用的(勾選)是大小寫英文、數字、特殊符號；最後一排按鍵分別是產生密碼、清除、複製最後一組密碼、全部複製：

其中已產生的密碼尚未清除之前會保留，如果重複按多次產生密碼鍵，就會累積，不會被覆蓋過去。

 

Crypto Notepad 的應用

Crypto Notepad 能做什麼呢？最簡單的當然是儲存「機密文字」，這裡的機密文字可能是帳密、可能是網址或其他不想讓別人知道的內容。搭配上次的 BitLocker 可以達到多層防護，或是互為防護的作用，另一個應用則是傳送機密內容給其他人。

設想一個情景：你把重要資料透過電郵、即時通訊傳給某人，當然電子郵件可以使用點對點加密(如 OpenPGP)，或是金鑰認證；很多即時通訊軟體也號稱全程加密，不過 Crypto Notepad 也能在此時使上力。

把重要資料透過 Crypto Notepad 存檔，整個文件傳輸，或是只發送經過加密的內文，只要跟對方約定好密碼、事先約定的編碼

約定的編碼舉例

  • 標題結束為阿拉伯數字，運算次數為 8487
  • 標題結束為中文，採 SHA384 + 10965 運算次數
  • 標題結束為英文，採 SHA1 + 20157 運算次數
  • 標題結束標點符號，運算次數為 26999
  • 以上為舉例，勿全數照抄...
方式，對方收到之後存在文字文件中，按照預先約定的編碼方式來解開訊息。下圖為加密後的內容意象圖：

📌KeePass 的安全設置要設多高？

KeePass 有許多設置跟安全相關，如果不想傷腦筋，直接用預設值就可以，或是如上面提供的做小修改。事實上 KeePass 還有一些設置，如鎖定的時機、演算法與次數、使用安全桌面...等。這些都可以增進安全性，不過便利與安全向來就無法兼顧，太講究提高安全性，肯定會加大使用上的不便。譬如閒置或最小化後鎖定，我使用不到一星期之後就統統關閉了，從 v1.x 用到現在 v2.45，這個設置再也沒打開過。為什麼呢？實在是太累人了！我的電腦也不會有其他人來使用，鎖定只是折磨到自己而已...

不過，使用安全桌面輸入主密碼可以算『物美(安全性高)價廉(不便增加不多)』的方法。打開安全桌面輸入之後，以後開啟 KeePass 就如同 UAC 控制下，執行管理員權限的時候，其他畫面都會暗下來，這樣可以最大程度保護不被攫取輸入的密碼。要啟用安全桌面輸入，請開啟 KeePass 的選項，在「安全」標籤下找到「在安全桌面輸入主鑰匙」，要啟用就勾選：

KeePass 二三事

KeePass 可講的地方其實還很多，我們提一下幾個重點：自動輸入、外掛、TAN 精靈。

很多人使用 KeePass 的原因是因為自動輸入，尤其是擁有大量帳密，需要輸入時使用 KeePass 比手動複製貼上快多了。不過凡事有利必有弊，自動輸入用久了，對帳號密碼也越來越模糊，甚是連帳戶名稱都記不全。這種現象在講真相時不足為奇，但是對於日常個人帳號就不太妙，我很不喜歡自己常用帳號都只知道模糊印象的幾個英文字，所以即使用 KeePass 記錄，我大多使用半自動，也就是帳號自己輸入，密碼從 KeePass 複製。

順便一提的是 KeePass 複製密碼後，預設在12秒之後就會清理剪貼簿，這是 KeePass 的安全機制，值得採用。大部分瀏覽器登入都能使用自動登入，甚至有段時間都可以登入QQ程式，可惜改版頻繁，現在又不行了。不管怎樣，自動登入對節省時間來說是大有幫助。有些使用自動輸入的注意事項：

  • 執行時輸入法請在英文模式
  • 快速鍵跟其他程式衝突就要修改
  • 有的網站會輸入會不正常，得手動去修改
  • 如果修改不成功，有時候外掛+擴充套件能搞定
  • 設定中有關自動輸入地方可視需要修改

KeePass 的外掛很多，使用時要注意區分 1.x 與 2.x 版。其實我個人是不太喜歡用外掛的，主要是沒外掛我就覺得滿好用了，另一個就是外掛是否安全？事關密碼，謹慎點準沒錯。雖說很多外掛也公開原始碼，不過我沒那麼多時間去好好解析，程式功力也不到家，就不用了吧...

如果有心研究，KeePass 官網提供很多資訊，除了外掛外還有很多平台的版本，如 Linux 版、手機板 (安卓、蘋果都有)。外掛如何安裝呢？下載之後解壓縮，放到 KeePass 程式所在的 Plugins 資料夾裡，從新開啟 KeePass 就可以在「工具」→「Plugin」中看到。


TAN(Transaction Authentication Number)精靈是用來記錄一次性認證碼的工具，很多服務開啟兩步驗證之後，會提供若干組一次性驗證碼，當手機壞掉、遺失、沒電...等情況時可以用來登入帳戶，這些驗證碼最好是不要存放在原認證裝置中，否則要用的時候一定很尷尬。

需要儲存一次性驗證碼的帳戶最好單獨放在一個群組中，因為TAN 精靈會把每組驗證碼視為一個項目，如果多個帳號在同一群組容易搞混。操作 TAN 精靈輸入時每一行填入一組驗證碼，像 Google 的驗證碼可能是8位數字，為了方便閱讀，當中會有一個空格，在建立時記得要刪除空格，否則會被視為兩組驗證碼。

以後每個驗證碼使用(或複製)之後，KeePass 都會標記為過期，歷史紀錄中也可以查看建立與使用的時間。如果有使用兩階段驗證，記錄備用驗證碼可以使用 KeePass 來幫忙。

 

KeePass 使用心得

從一開始純粹用來記憶密碼，到記錄一些私人資訊，如各種卡申辦、到期日，甚至家裡電冰箱哪時候買的，這種瑣碎但似乎有時候會用到的訊息都記錄起來。反而正規的自動輸入之類的功能我很少用，幾乎每天需要用到的密碼也很少放在 KeePass，這裏面放的其實是不常用到，但是不見了會跳腳的帳密、資訊。

當然這僅僅是我個人的習慣，常用的訊息反而是記在 Crypto Notepad 之內的軟體。後來發現把講真相一些帳密存在 KeePass 很方便，直接放在講真相虛擬機，除了帳密、網址相關訊息之外，還能運用備註欄位記錄其他事項，甚至還能附加檔案、增加自訂欄位。用來管理一堆講真相帳號其實滿好用的，比用 Excel 更好，安全性更是大大提高！

KeePass 使用主密碼 + 金鑰檔來防護應該夠了，程式也有結合 Windows 帳戶的功能，但是不推薦使用，因為從某個角度來講，多一層防護有時候是多一個枷鎖，而且常常都是鎖住自己。比較重要的是資料庫、主密碼與金鑰檔的備份，這三個哪一個不見了都是災難。外接硬碟、雲端、另一台電腦都是可以考慮的地方。

順便提一下：備份的時候這三者不要放在一起，可以的話多用一層防護(尤其放雲端)。另外，KeePass 的資料庫其實可以偽裝，這也是可以利用的地方。


 

徹底刪除重要資料

資訊安全另一個課題：安全銷毀。這是個不容易遇到，但是遇到就很致命的問題。一般人電腦資料刪除就刪除了，沒什麼安全銷毀的問題。但是企業，如重要或關鍵技術一旦被竊取，那可是大事。即使是個人，養成把重要文件徹底銷毀是個好習慣。

把資料從硬碟刪除，實際存放的資料並沒有真正刪掉，如果遇到有心人，還是有可能被讀取。當然這要符合若干條件才會成立，對個人來講，花點時間用一些小工具把不要的重要資料刪乾淨一點應該是值得的。

隱藏檔案或資料夾

隱藏檔案或資料夾是個另類需求。原本 Windows 會有隱藏選項的設置，是怕使用者把重要的系統檔案刪除了而造成不幸，不過這也能拿來用在隱藏重要資料上。

把重要資料隱藏起來最怕的，就是自己忘了...不誇張，因為隱藏的資料可能不常用，日子久了一不小心刪除了或是忘記放哪了都有可能。好吧！這是我自己的血淚經驗，提出來警世一下。

要隱藏檔案可以用系統內建的方式也可以用第三方工具。前者易用也易破解，純粹是防君子用的；後者可選擇較多，但是好用又免費者並不多，這裡介紹兩種。

Windows 本身可以設置隱藏檔案，最簡單的方式是對檔案按右鍵，在內容頁籤中把隱藏屬性勾選即可，要進一步可以用 attrib 命令來操作。不過這種方式都很容易就顯示出來，我們希望的隱藏是兼具保護作用。

簡單的資料夾鎖 IsyLock

IsyLock 是個很有意思的軟體，很單純的用密碼鎖定特定資料夾然後隱藏，沒有其他設定。

IsyLock 小檔案

  • 官網：https://sourceforge.net/projects/isylock/
  • 下載：到官網點 Download
  • 特點：免安裝，極簡，另類隱藏風格的軟體。

 

下載 IsyLock 解壓縮存放好之後別急著執行，請先了解 IsyLock 的運作機制。

複製 IsyLock.exe 到想要隱藏資料的地方，執行 IsyLock，輸入二次密碼(可以用含中文的強式密碼)：

 

IsyLock 執行的時候會在程式所在的位置自動建立一個「LockFolder」資料夾，請把要隱藏的資料都放到 LockFolder 中，然後點程式右邊的鎖，點了之後鎖頭會鎖上，那個 LockFolder 資料夾就隱藏起來了。如下圖：

 

看到這裡懂了吧！IsyLock 所在的位置就是要隱藏資料夾的地方，被隱藏的資料夾就是 LockFolder，不能是其他名稱。隱藏資料夾之後要偽裝還可以放幾個其他檔案，把 IsyLock 移走，要解鎖的時候再把 IsyLock 移回來執行。

這樣的特性可以到處隱藏資料夾，也可以使用不同的密碼，前提是自己要記住哪些地方隱藏了、用了哪些密碼？保留一個 IsyLock 放在沒有隱藏的地方就可以。

IsyLock 隱藏的資料夾其實在開啟含顯示系統隱藏檔的時候可以看到，會有一個類似資源回收筒的系統文件，不過也無法讀取。

IsyLock 跟 Folder Password Lock Free 各有特色，請自己斟酌取用吧！

這類工具可以搭配 BitLocker 或 KeePass 使用，例如把 BitLocker 建立的 VHD 檔案隱藏起來，或是搭配 KeePass 記憶路徑與密碼。總之變化之妙，存乎一心。
